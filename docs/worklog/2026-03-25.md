# 수정 이유
- 기존 NCP 도메인이 사라진 상태에서 리다이렉트와 초대 링크가 모두 실패할 수 있었음
- 로컬 Ubuntu Docker 배포 환경에 맞게 주소를 설정 파일로 바꿔야 환경별 전환이 가능함
- CORS 허용 Origin도 코드 수정 없이 설정만으로 변경할 수 있어야 유지보수가 쉬움
- 프런트 운영 환경변수도 이전 NCP 도메인을 바라보고 있어 현재 로컬 nginx 진입점 기준으로 교체가 필요했음
- OAuth callback도 `localhost:8080` 고정값이면 현재 nginx 단일 진입점(`127.0.0.1`)과 불일치할 수 있어 별도 설정으로 분리했음
- `spring-ai-starter-model-google-genai` 만으로는 `EmbeddingModel` 빈이 생성되지 않아 Google GenAI embedding starter를 추가함

# 추가로 해야 할 작업
- 프런트 `VITE_API_BASE_URL` 을 현재 nginx 라우팅 구조에 맞게 `http://127.0.0.1/api` 기준으로 점검
- OAuth provider에 등록된 redirect URI가 현재 로컬 배포 주소와 일치하는지 확인
