## 수정 이유
- 챗봇 API 진입점과 RAG 문서 자동 적재 빈이 활성화된 상태에서 AI 관련 설정 이슈로 앱 부팅을 방해할 수 있어 최소 수정으로 비활성화했다.
- NCP Object Storage 기본 주소 하드코딩을 제거하고 MinIO 기반 이미지 주소를 프런트 환경변수로 관리하도록 정리했다.
- `deskit.conf` 와 `default.conf` 가 동시에 `listen 80`, `server_name _` 로 활성화되어 있어 `/api/broadcasts/subscribe/all` 요청이 의도한 SSE 프록시 설정이 아닌 다른 서버 블록으로 매칭될 수 있었다.
- 백엔드 `127.0.0.1:8080` 직접 호출에서는 SSE `connect` 이벤트가 즉시 내려왔고, nginx 경유 `127.0.0.1/api/...` 호출에서는 바이트가 전송되지 않아 프록시 충돌로 판단했다.
- 사용하지 않는 `default.conf` 를 비활성화한 뒤 SSE 구독이 정상 연결되어 `504 Gateway Time-out` 문제가 해소됐다.
- 워크로그와 대화 기록 Markdown 파일들 사이에 UTF-8 BOM 유무가 섞여 있었고, Windows PowerShell 코드페이지 환경에서 한글이 자주 깨져 보여 문서 폴더 인코딩 정책을 `utf-8-bom` 으로 통일했다.
- `.editorconfig` 를 추가해 향후 문서 저장 시 편집기마다 인코딩이 달라지는 문제를 줄였다.

## 추가로 해야 할 작업
- 백엔드 이미지를 재빌드하고 `deskit-app` 컨테이너를 재기동한 뒤 최신 Docker 로그를 다시 확인
- 챗봇 기능 재활성화 시 주석 해제 대신 설정 분리 방식으로 토글 가능하게 정리
- nginx 운영 설정은 하나의 server 블록만 기본 진입점으로 유지하고, 비활성화한 conf 파일은 재활성화되지 않도록 관리
- 프런트 정적 파일과 nginx 설정 변경 사항을 배포 절차 문서에 반영
- 필요하면 PowerShell 프로필에도 UTF-8 코드페이지 설정을 추가해 콘솔 출력 인코딩까지 고정
