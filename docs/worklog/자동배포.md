# 자동 배포 작업 정리

작성일: 2026-03-28

## 작업 목적
- Windows 프로젝트 경로에서 백엔드와 프론트 산출물을 생성한다.
- 생성된 산출물을 WSL Ubuntu의 지정 경로로 복사한다.
- 백엔드 이미지를 `deskit:latest` 로 빌드하는 자동화 흐름을 정리한다.

## 사용자 요청 요약
- WSL Ubuntu `root` 계정에 접근 가능한지 확인
- 현재 프로젝트에서 백엔드 JAR 생성 가능 여부 확인
- 현재 프로젝트에서 프론트 빌드 생성 가능 여부 확인
- 생성된 파일을 Ubuntu 경로로 복사 가능한지 확인
- 백엔드는 Docker 이미지 빌드까지 포함한 자동화 구성
- 위 내용을 문서화하고 실제 실행 스크립트까지 구성

## 확인한 환경
- 프로젝트 루트: `C:\personal_project\refactoring\deskit`
- 프론트 경로: `C:\personal_project\refactoring\deskit\front`
- WSL 배포판: `Ubuntu`
- WSL `root` 계정 진입 가능 확인 완료
- WSL에서 `/mnt/c` 경로 접근 가능 확인 완료

## 확인한 빌드 방식
### 백엔드
```powershell
.\gradlew.bat bootJar
```

- 산출물 경로:
```text
build/libs/deskit-0.0.1-SNAPSHOT.jar
```

### 프론트
```powershell
npm run build
```

- 산출물 경로:
```text
front/dist
```

### 백엔드 이미지
- 프로젝트 루트에 별도 `Dockerfile` 은 없었음
- 따라서 Spring Boot `bootBuildImage` 기반으로 자동화 구성

```powershell
.\gradlew.bat bootBuildImage --imageName=deskit:latest
```

## 확정된 배포 값

- 백엔드 JAR 배치 경로: `/srv/livecommerce/`
- Docker 이미지 이름: `deskit`
- Docker 태그: `latest`


처리 순서:
1. 백엔드 JAR 생성
2. 프론트 정적 파일 빌드

## 작업 중 발생한 문제
### 1. `npm` 호출 오류
초기 스크립트 실행 시 아래 오류가 발생함.

```text
Unknown command: "pm"
```

원인:
- PowerShell 스크립트 내부에서 `npm.ps1` 이 인자를 비정상적으로 해석함

조치:
- `npm` 대신 `npm.cmd` 를 직접 호출하도록 스크립트 수정

수정 내용:
```powershell
$npmCommand = (Get-Command npm.cmd -ErrorAction Stop).Path
& $npmCommand run build
```

### 2. `spawn EPERM` 오류
재검증 중 아래 오류가 발생함.

```text
Error: spawn EPERM
```

원인:
- 현재 에이전트 실행 환경의 권한 제한으로 `vite/esbuild` 자식 프로세스 실행이 차단됨

판단:
- 사용자 로컬 환경에서는 스크립트가 정상 실행될 가능성이 높다고 판단

결과:
- 사용자 환경에서 실제 실행 성공 확인

## Ubuntu 운영 Docker 기준 확인
### 확인한 파일
- `/srv/livecommerce/Dockerfile`
- `/srv/livecommerce/docker-compose.yml`

### 확인 결과
- `Dockerfile` 은 `deskit-0.0.1-SNAPSHOT.jar` 를 복사해 실행하는 구조였다.
- `docker-compose.yml` 은 `deskit:latest` 이미지를 사용하고 있었다.
- 따라서 운영 기준은 `bootBuildImage` 보다 Ubuntu 경로에서 `docker build -t deskit:latest .` 를 수행하는 방식이 더 적절하다.

## 자동화 스크립트 조정
기존:
```powershell
.\gradlew.bat bootBuildImage "--imageName=deskit:latest"
```

변경 후:
```powershell
wsl.exe -d Ubuntu -u root -- sh -lc "cd /srv/livecommerce && docker build -t deskit:latest ."
```

## WSL 폴더를 직접 여는 방법
자동화와 별도로, Ubuntu 파일을 Windows 탐색기에서 직접 확인할 수도 있다.

### 탐색기 주소창으로 열기
파일 탐색기 주소창에 아래 경로를 입력하면 된다.

```text
\\wsl$\Ubuntu\
```

바로 대상 폴더를 열고 싶으면 다음 경로를 사용한다.

```text
\\wsl$\Ubuntu\srv\livecommerce
\\wsl$\Ubuntu\var\www\deskit
```

### PowerShell에서 탐색기로 열기
```powershell
explorer.exe \\wsl$\Ubuntu\srv\livecommerce
explorer.exe \\wsl$\Ubuntu\var\www\deskit
```

### 의미
- 이 방식은 `wsl.exe` 로 셸 명령을 실행하는 방식이 아니라, Windows 탐색기에서 WSL 파일시스템을 네트워크 경로처럼 여는 방식이다.
- 따라서 `Dockerfile`, `docker-compose.yml`, `jar` 파일, 프론트 정적 파일을 폴더 형태로 직접 확인할 수 있다.

### 주의사항
- `\\wsl$\Ubuntu\` 경로는 WSL 배포판이 실행 중일 때 더 안정적으로 열린다.
- 시스템 상태에 따라 처음 접근 시 다소 느릴 수 있다.
