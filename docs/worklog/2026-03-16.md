# 작업 목적
- OpenAI 기반 AI 호출을 Gemini 기반 Spring AI 설정으로 전환

# 수정한 파일
- `build.gradle`
- `src/main/java/com/deskit/deskit/ai/chatbot/openai/service/OpenAIService.java`
- `src/main/java/com/deskit/deskit/ai/chatbot/rag/service/RagService.java`
- `src/main/java/com/deskit/deskit/ai/evaluate/service/SellerPlanEvaluationService.java`
- `src/main/java/com/deskit/deskit/ai/config/EmbeddingConfig.java`
- `src/main/resources/application.properties`
- `.gitignore`

# 수정 전 코드
```gradle
implementation 'com.openai:openai-java:4.13.0'
implementation("com.squareup.okhttp3:okhttp:4.12.0")
```

```java
@Value("${spring.ai.openai.chat.options.model}")
private String chatModel;

private final OpenAIClient openAIClient;
```

```properties
spring.ai.openai.api-key=...
spring.ai.openai.chat.options.model=gpt-4o-mini
spring.ai.openai.embedding.options.model=text-embedding-3-small
```

# 수정 이유
- OpenAI SDK 직접 호출을 제거해야 Gemini 공급자 변경 시 서비스 로직을 유지할 수 있음
- Spring AI 공통 인터페이스를 사용하면 채팅/임베딩 구현체를 공급자에 덜 결합된 상태로 유지할 수 있음
- `.env`를 Git 추적 대상에서 제외해 이후 키 분리를 안전하게 진행할 수 있음

# 추가로 해야 할 작업
- Gemini API 키를 `.env` 또는 실행 환경변수로 주입
- `application.properties`의 남아 있는 민감정보를 외부 설정으로 분리
- 실제 `./gradlew test` 또는 `./gradlew bootJar`로 빌드 검증

# 추가 대화 정리
- `spring.ai.google.genai.api-key`는 Gemini 채팅/생성 모델 호출용 기본 키로 이해하면 됨
- `spring.ai.google.genai.embedding.api-key`는 텍스트를 벡터로 변환하는 임베딩 요청에만 별도로 사용할 수 있는 키임
- 임베딩은 RAG, 유사도 검색, 벡터 DB 저장을 위해 문장을 숫자 벡터로 바꾸는 용도임
- 일반 채팅 키와 임베딩 키의 차이는 호출 대상 기능의 차이이며, 초기 단계에서는 하나의 Gemini 키만으로도 운영 가능한 경우가 많음
- 비용 분리, 권한 분리, 배치 서버 분리 같은 운영 목적이 있을 때만 임베딩 전용 키를 별도로 두는 구성이 유리함
