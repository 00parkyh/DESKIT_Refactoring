## 수정 후 코드
```powershell
param(
    [string]$BackendTargetDir = "/srv/livecommerce",
    [string]$FrontendTargetDir = "/var/www/deskit",
    [string]$ImageName = "deskit",
    [string]$ImageTag = "latest"
)

& .\gradlew.bat bootJar
& npm run build
& .\gradlew.bat bootBuildImage "--imageName=$ImageName:$ImageTag"
& wsl.exe -d Ubuntu -u root -- sh -lc "mkdir -p '$BackendTargetDir' '$FrontendTargetDir'"
```

## 수정 이유
- 사용자가 원하는 배포 흐름은 단순 문서화가 아니라 실제 반복 가능한 자동화 실행이었다.
- 사용자가 기존에 사용하던 `docker build -t deskit .` 는 태그 생략 형태이므로 기본 태그를 `latest` 로 고정하는 것이 가장 자연스럽다.
- 현재 프로젝트에는 `Dockerfile` 이 없기 때문에, 이미지 빌드는 `bootBuildImage` 로 처리하도록 자동화했다.
