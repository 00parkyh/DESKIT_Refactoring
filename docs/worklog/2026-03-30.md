```md
## 작업 목적
- 구글 로그인/회원가입 시 표시되는 `액세스 차단됨` 원인을 컨테이너 로그와 OAuth 설정 기준으로 확인

## 수정 이유
- `deskit-app` 컨테이너 로그에는 구글 OAuth 콜백 이후의 애플리케이션 예외가 확인되지 않았다.
- 프론트는 `VITE_OAUTH_BASE_URL` 이 없으면 현재 브라우저 호스트 기준으로 `/oauth2/authorization/google` 로 이동한다.
- 백엔드는 `deskit.web.base-url` 이 없으면 `http://127.0.0.1` 을 기본값으로 사용한다.
- 현재 컨테이너 환경에는 구글 OAuth 클라이언트 값은 있으나 `deskit.web.base-url` 과 명시적 redirect URI 설정은 보이지 않았다.
- 따라서 이번 증상은 애플리케이션 내부 예외보다 Google Cloud Console 의 OAuth 승인 도메인 / Authorized redirect URI / 테스트 사용자 제한 불일치 가능성이 높다.
```

## 수정 이유
- 사용자 요청은 컨테이너 로그 확인이었고, 실제로 `deskit-app` 로그에서 애플리케이션 내부의 OAuth 예외는 재현되지 않았다.
- 코드와 컨테이너 환경변수를 함께 확인한 결과, 구글 측 선차단 가능성을 후속 조치 대상으로 남길 필요가 있었다.

## 추가로 해야 할 작업
- Google Cloud Console 에 등록된 `Authorized JavaScript origins` 와 `Authorized redirect URIs` 를 실제 접속 주소 기준으로 대조
- 필요하면 `deskit.web.base-url`, `deskit.cors.allowed-origins`, `VITE_OAUTH_BASE_URL` 을 동일한 호스트 기준으로 명시
